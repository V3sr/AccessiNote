# Demo Script

AccessiNote demo target: Microsoft Agents League Creative Apps + Accessibility.

## Five-Minute Flow

1. Open the local app and show that no account, cloud database, or API key is required.
2. Load the sample lecture to show the baseline evidence timeline and output modes.
3. Upload a short permitted lecture video.
4. Call out the staged processing flow: upload received, transcription, visual change detection, OCR, alignment, ready for review.
5. Review the scan report: candidate frames, selected frames, OCR frames, caption source, weak chunks.
6. Inspect the timeline: keyframes, transcript text, OCR text, visual review notes, concepts, confidence, evidence flags.
7. Generate Screen Reader Notes and show visual descriptions and source coverage.
8. Generate WebVTT Captions and download the `.vtt` file.
9. Generate Evidence JSON to show transparent source grounding.
10. Close with the safety posture: permitted materials only, human review required, local-first processing.

## Judging Alignment

- Accuracy and relevance: timeline chunks preserve transcript, OCR, keyframes, and confidence.
- Reasoning and multi-step thinking: local pipeline extracts audio, detects visual changes, scans OCR, aligns evidence, and flags weak chunks.
- Creativity and originality: lecture accessibility is treated as multimodal evidence review, not just summarization.
- User experience and presentation: the workbench shows progress, scan metrics, source references, and export controls in one flow.
- Reliability and safety: no required external API keys, explicit warnings, local storage, and human-in-the-loop review.
- Accessibility: screen-reader notes, captions, plain-language output, ADHD study packs, and notetaker quality reporting are first-class outputs.

## Demo Data Guidance

Use synthetic or explicitly permitted lecture material. Avoid private student data, exams, accommodation records, or copyrighted lecture recordings without permission.
